# Release Notes `v1.4.0`

Fecha: 2026-02-27

## Resumen
Versión estable de liquidación y detalle dashboard para requisiciones internas, manteniendo el stack simple (FastAPI + SQLite + SSR).

## Incluye
- Flujo de liquidación completo (`entregada` -> `liquidada`) con validaciones operativas y alertas no bloqueantes.
- Modal de detalle en formato dashboard con timeline, alertas de conciliación y tabla liquidada.
- Soporte operativo para `prokey_ref` pendiente y edición posterior por usuario autorizado.
- Lógica por ítem `RETORNABLE/CONSUMIBLE` con diferencia y alertas consistentes.
- Bloqueo de liquidación para ítems entregados sin definición (todo en cero).
- Mejora UX final:
  - `Alta severidad` en vez de `Severidad alta`.
  - Acción sugerida según tipos de alerta.
  - `Ingreso PK` como `—` para consumibles.
  - Columnas numéricas centradas.
  - Botón `Ver PDF` habilitado solo al estar liquidada.
- Corrección de hora de creación en nuevas requisiciones:
  - `created_at` se persiste en hora local para evitar desfase de +6h en detalle.

## Validación
- Tests de liquidación ejecutados en `main`: `32 passed`.
